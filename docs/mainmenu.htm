Main Menu

File Menu

The file menu contains most of the file input/output related functions of the Behaviour Tool.

  • New Behaviour (Ctrl+N)
    • Clears out the information from the current behaviour and starts a new one. The user will always be prompted to save the current behaviour, as once you start a new behaviour, you can't retrieve the changes made to the old behaviour unless it was saved.
  • Open Behaviour... (Ctrl+O)
    • Opens a generated behaviour file for editing in Behaviour Tool. Behaviour Tool gets its information about the behaviour by examining its opening comment block. For this reason, changes that an author makes to a behaviour manually, after it has been generated, is not maintained by the Behaviour Tool. Authors should note their changes before opening a behaviour in the tool, as it is easy to accidentally save a newly generated behaviour over the old, manually edited one. Like starting a new behaviour, the user will always be prompted to save the current behaviour before opening an existing behaviour.
  • Save Behaviour (Ctrl+S)
    • Saves a copy of the current behaviour. Upon activating this menu item, the current working directory (that is, the last directory you opened or saved a file in, or the directory containing the Behaviour Tool executable) is searched for previously generated script files matching the current behaviour name. If those files are found, then the tool assumes this directory is the correct directory, and saves the two script files to this directory. If the files are not found, a dialog box will pop up asking the user to select the directory they would like to save the script files in.
  • Save Behaviour In... (Alt+S)
    • Saves a copy of the current behaviour in a specific directory. Unlike the Save Behaviour menu item, with Save Behaviour In, the user is always prompted to select the directory they would like to save the script files in.
  • Exit
    • Exits the behaviour tool. The user will always be prompted to save the current behaviour, as exiting without saving changes will lose the changes made to the current behaviour.

Edit Menu

The edit menu contains methods useful for entering and manipulating text in the various editable fields on the interface.

  • Undo (Ctrl+Z)
    • Returns the field to the previous state. For example, if you accidentally delete the text in a text field, selecting this menu item (or pressing the keyboard shortcut) will undo the deletion, returning the text to its previous state.
  • Redo (Ctrl+Y)
    • Returns the field to the state previous to the last undo action. Extending the previous example, if you find that you did indeed want to delete the text in that text field, selecting thie menu item (or pressing the keyboard shortcut) will redo the deletion, removing the text from the field.
  • Cut (Ctrl+X)
    • Copies the selected text to the clipboard and removes it from the text field.
  • Copy (Ctrl+C)
    • Copies the selected text to the clipboard without removing it from the text field.
  • Paste (Ctrl+V)
    • Copies the text in the clipboard to the text field at the current cursor position. If other text is selected, it will remove the selected text and replace it with the text in the clipboard.

Tools Menu

The tools menu contains functions that do not need to be used often, but are still very useful.

  • Options...
    • Selecting this menu item brings up the options dialog. In its current state, the only option is to store the location of the util_verbs.nss file. Doing this will load up the file when the tool starts up so that you don't have to re-open it every time the program starts up.

Help Menu

The help menu contains funtions to help the user use the tool.

  • Help Contents (F1)
    • Launches this, the help file! The help file can also be loaded up on its own by running the behaviour_tool.chm file in the Behaviour Tool directory.
  • About Behaviour Tool
    • Provides some information about the tool and its developer. Further information can also be found on the about page of this help file.
