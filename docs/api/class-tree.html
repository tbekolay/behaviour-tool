Class Hierarchy

  • object: The most base type
    • wx.lib.ogl._basic.ShapeEvtHandler
      • Gui.VerbShapeHandler: Enables us to change tabs when a verb on the VerbCanvas is double-clicked.
    • wx._core.Object: The base class for most wx objects, although in wxPython not much functionality is needed nor exposed.
      • wx._core.EvtHandler: Proxy of C++ EvtHandler class
        • wx._core.Window: wx.Window is the base class for all windows and represents any visible object on the screen.
          • wx._core.Control: This is the base class for a control or 'widget'.
            • wx._controls.ListCtrl: Proxy of C++ ListCtrl class
              • Gui.ActorListCtrl: The ActorListCtrl is the widget that actors will be inputted into.
              • Gui.NWVarListCtrl: The NWVarListCtrl is the widget that NWVariables will be inputted into.
            • wx._controls.BookCtrlBase: Proxy of C++ BookCtrlBase class
              • wx._controls.Notebook: Proxy of C++ Notebook class
                • Gui.MainNoteBook: The notebook that encompasses the rest of the interface.
            • wx.stc.StyledTextCtrl: Proxy of C++ StyledTextCtrl class
              • Gui.NWScriptSTC: A styled text control (a wxWidget based on Scintilla) for the display of NWScript code.
          • wx._windows.TopLevelWindow: Proxy of C++ TopLevelWindow class
            • wx._windows.Dialog: Proxy of C++ Dialog class
              • Gui.BOptionsDialog: BOptionsDialog is the dialog that we'll use to set various options in the config object.
            • wx._windows.Frame: Proxy of C++ Frame class
              • Gui.MainFrame: The main frame for the program, containing a menu and the notebook.
          • wx._windows.Panel: Proxy of C++ Panel class
            • Gui.BehaviourCodePreviewPanel: A panel containing widgets to select between two choices of viewable code, and preview it.
            • Gui.BehaviourPanel: This panel contains widgets to display and modify behaviour-wide information.
            • Gui.VerbPanel: This panel contains widgets to view and modify a verb.
            • wx._windows.ScrolledWindow: Proxy of C++ ScrolledWindow class
              • wx.lib.ogl._canvas.ShapeCanvas
                • Gui.VerbCanvas: The canvas that the verb graph will be drawn on.
            • Gui.VerbCodePreviewPanel: A panel containing widgets to view two code preview widgets simultaneously.
          • wx._windows.SplitterWindow: wx.SplitterWindow manages up to two subwindows or panes, with an optional vertical or horizontal split which can be used with the mouse or programmatically.
            • Gui.BehaviourSplitter: Allows the user to vary how much they see of the two sections of a behaviour tab.
            • Gui.VerbSplitter: Allows the user to vary how much they see of the two sections of a verb tab.
        • wx._core.PyApp: The ``wx.PyApp`` class is an *implementation detail*, please use the `wx.App` class (or some other derived class) instead.
          • wx._core.App: The ``wx.App`` class represents the application and is used to: * bootstrap the wxPython system and initialize the underlying gui toolkit * set and get application-wide properties * implement the windowing system main message or event loop, and to dispatch events to window instances * etc.
            • Gui.BehaviourApp: Our program.
    • Codegen.Behaviour: An object to hold all of the information necessary to generate the code for a behaviour.
    • exceptions.BaseException: Common base class for all exceptions
      • exceptions.Exception: Common base class for all non-exit exceptions.
        • Parser.ParseError: An exception raised by the parsing functions when the input script is not well-formed.
    • Codegen.ActualVerb: ActualVerb holds information about an 'actual verb'.
    • Codegen.NWVariable: Contains information about an NWScript variable.
    • Codegen.Verb: Verb generates code for each verb object.
    • Gui.Model: The Model keeps track of all of the data that the GUI needs.
  • wx._core.PyOnDemandOutputWindow: A class that can be used for redirecting Python's stdout and stderr streams.
  • wx.lib.mixins.listctrl.ListCtrlAutoWidthMixin: A mix-in class that automatically resizes the last column to take up the remaining width of the wx.ListCtrl.
    • Gui.ActorListCtrl: The ActorListCtrl is the widget that actors will be inputted into.
    • Gui.NWVarListCtrl: The NWVarListCtrl is the widget that NWVariables will be inputted into.
  • wx.lib.mixins.listctrl.TextEditMixin: A mixin class that enables any text in any column of a multi-column listctrl to be edited by clicking on the given row and column.
    • Gui.ActorListCtrl: The ActorListCtrl is the widget that actors will be inputted into.
    • Gui.NWVarListCtrl: The NWVarListCtrl is the widget that NWVariables will be inputted into.
