Behaviour Tab

The behaviour tab is the first tab in the Behaviour Tool interface. This tab is always present, and cannot be closed or hidden. The tab is split into two sections, one in which you can edit behaviour wide objects, and one in which you can view the code that the current behaviour will produce when saved.

Fields

The following fields are all encompassed in the Behaviour panel; a panel containing widgets to edit behaviour wide objects.

  • Behaviour name Text field
    • This text field allows the author to change the name of the behaviour. Spaces are not allowed in the behaviour name because the behaviour name is used in variable names and the script name; any spaces will be converted to underscores.
    • The change in behaviour name happens when the focus leaves the text field. Once the behaviour changes, so does the text on the behaviour tab and the dropdown box in the Behaviour Code Preview panel.
  • Actors List control
    • This list control allows authors to define the actors that will be used in the behaviour. To add a new actor, press the + button. To edit the name and description of the actor, click on the text that you would like to change, and type in a new value. If you wish to delete an actor, click or press the arrow keys until the entire row is selected, then press the - button. If you decide that you deleted an actor prematurely, or accidentally, you can restore a deleted actor by pressing the Trash button, then selecting the actor that you wish to restore.
    • Validation
      • Values in the Name field must be of the form o<name, with first letter capitalized> with no spaces; for example, oVictim. Text that does not fit this form will be changed to fit it automatically.
      • Values in the Description field must be non-blank. Attempting to blank out the description field will change the description back to its default value "Description".
  • Other Variables List control
    • This list control allows authors to define the other variables that will be used in the behaviour. Other variables would be things like strings a character might say, or a location that the chracter will move to. To add a new varaible, press the + button. To edit the type, name and description of the variable, click on the text that you would like to change, and type in a new value. If you wish to delete a variable, click or press the arrow keys until the entire row is selected, then press the - button. If you decide that you deleted a variable prematurely, or accidentally, you can restore a deleted variable by pressing the Trash button, then selecting the variable that you wish to restore.
    • Validation
      • Values in the Type field must be one of "object", "string", "int", or "float". This list can be added to. Any value not in the list will not be accepted, and the type will revert to its default, currently "object".
      • Values in the Name field must be of the form <first letter of the type><name, with first letter capitalized> with no spaces; for example, sConv. Text that does not fit this form will be changed to fit it automatically.
      • Values in the Description field must be non-blank. Attempting to blank out the description field will change the description back to its default value "Description".
  • Verbs Graph
    • As the number of verbs and connections between verbs increases, it can be difficult to visualize the web of verbs and their relationships. This graph gives the author one way of aiding the visualization, by generating a graph showing all of the verbs and their relationships. Each verb has an arrow pointing to their followers. One can move the verbs around to make the most effective graph. Double clicking on a verb will take you to that verb's tab. One can add another verb by pressing the + button at the top right. Deleted verbs can be restored by pressing the Trash button and selecting the desired verb.

The following fields are encompassed in the Behaviour code preview panel; a panel containing widgets to view the code that the current behaviour will produce when saved.

  • Preview Generated Code Dropdown box
    • This dropdown lets you choose which script file you want to preview, the b_<behaviour name>.nss or z_b_<behaviour name>.nss file. After you add an actor or variable, you can also pretty the refresh button to update the code preview.
  • Generated Code Styled text control
    • This readonly text control allows the author to see the code that the Behaviour Tool has produced. It supports syntax highlighting and code folding by clicking on the + and - boxes in the margin.
